Introduction

I always had a challenge with; when I find a solution for a certain problem and months down the line I run into a very similar problem and try to remember how I solved the previous one. So as many professionals in the IT business I created a blog to write those solutions down for myself and anyone who stumbles upon the blog. Unfortunately my blog at that time has followed the trend of the general population of IT related blogs and withered away due to the lack of attention from yours truly.

The need to store the solution to a certain problem however has always been present for me. However the required effort for maintaining a blog usually via a CMS like system with extensions and add-ins for various features has always refrained me from starting another blog. In addition to the maintenance effort the tools required for writing a blog post where also kind of as a threshold.

This however changed last year when I discovered that tool exist which can generate static content from Markdown. So what’s the big change about you ask?

I can use my dev & scripting tools like VS Code Markdown is a language I am quite familiar with Posting is done

-> Markdown, low maintenance, fire and forget, using a method that I am very familiar with.

Which eventually lead to me creating the current blog. Therefore its seems fitting to me to cover setting up such a static content generation blog in my first post.
